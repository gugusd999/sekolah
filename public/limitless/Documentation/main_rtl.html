  • Home

RTL layout overview
Overview

RTL layout - layout with right-to-left direction, where content displays from the right-hand side of a page and concludes on the left-hand side, such as in Arabic, Hebrew and Urdu. Other words - the whole page layout and content is mirrored. Limitless template supports RTL layout version in all 4 available main layout variations.

Screenshot of RTL folder

The difference

Both LTR and RTL versions use the same markup, class names and content ordering. The only difference is dir="rtl" attribute added to the <html> element of the page, which changes the direction. However, LESS/CSS files are totally different and some JS files include special configuration options for specific RTL usage.

Also some of the plugins are modified in order to work with RTL direction, because by default they don't support it. A couple of components, such as NoUI slider and 2 Datatable extensions don't support RTL at all due to the plugin logic, but NoUI sliders will be fixed soon. All other options and features are supported.

Example markup:

<!-- RTL layout -->
<html lang="ar" dir="rtl">

	<!-- Page head -->
	<head>
		...
	</head>
	<!-- /page head -->


	<!-- Page body -->
	<body>

		<!-- Main navbar -->
		<div class="navbar navbar-inverse">
			...
		</div>
		<!-- /main navbar -->


		<!-- Page container -->
		<div class="page-container">
			...
		</div>
		<!-- /page container -->

	</body>
	<!-- /page body -->

</html>
<!-- /RTL layout -->
Bootstrap RTL

By default, Bootstrap framework doesn't support RTL direction. But fortunately it can be updated manually. Limitless template uses a custom manually updated Bootstrap LESS version, which is the same as LTR version, except a couple of additinal fixes. Table below contains files that belongs to original Bootstrap LESS files and have been modified:

What Where Description
grid-framework.less /less/bootstrap/mixins/ Changed floating of columns and horizontal spacing
border-radius.less /less/bootstrap/mixins/ Changed direction of horizontal border radius

Almost all other LESS files have been updated as well, so LESS files from LTR layout versions can't be used with RTL layout, because most of them have different direction.

Rules and notices

All components are adapted for use in RTL version, both LESS files and JS files. Since in RTL layout everything looks like in mirror, all classes remain the same, but do the opposite: .border-left-lg changes left border in LTR version and right border in RTL version, while .border-right-lg changes right border in LTR version and left border in RTL version. This method allows us to use the same markup in all possible contexts and languages, the only difference is CSS files.

Also please keep in mind, that core bootstrap.css file needs to be included from the package, but not from the official Bootstrap website, because core mixin that builds grid system has been modified, so we need to use custom built Bootstrap version.

This documentation works for both layout directions: all components, libraries, plugins and extensions are the same.
